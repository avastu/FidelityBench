## Key Observations

- `WindowedTranscriptLLMAgent` led this single run overall with 383 total, followed by `BlockMemoryLLMAgent` at 373, `TranscriptLLMAgent` at 369, and `HybridGraphSemanticMemoryLLMAgent` at 367.
- `GraphMemoryLLMAgent` was strongest on relational/reflection-style tasks: 100/100 on board privacy, 108/108 on reflection, and 95/100 on Alex pushback.
- `HybridGraphSemanticMemoryLLMAgent` matched graph memory on board and Alex, and was close on reflection, but lost points on temporal supersession and dinner.
- The graph-only baseline exposed a meaningful action gap: it often preserved intent in prose but failed to call restaurant tools on dinner and temporal scenarios.
- The Alex scenario is now a cleaner graph-memory showcase: graph, hybrid, and block memory reached 95/100 under neutralized prompts and shared response instructions; transcript/windowed transcript reached 90/100.
- Shared response instructions reduced bespoke task coaching, but tool-loop completion remains weak across LLM baselines.

## Failure Modes

- Tool-loop incompletion: no LLM baseline completed a successful restaurant hold in dinner or temporal supersession.
- Graph action gap: `GraphMemoryLLMAgent` did not call restaurant tools in dinner or temporal supersession despite retaining some relevant intent.
- Hybrid stale retrieval: `HybridGraphSemanticMemoryLLMAgent` retrieved/acted on stale or incorrect context in temporal supersession, including an Italian/Bella Tavola-shaped search.
- Directness heuristic misses: on Alex pushback, most high-performing LLM agents named risk and tradeoff but lost the direct-communication dimension.
- Recall burden persists in small ways: several LLM agents asked for known time, dietary, or generic clarification information.

## Limitations

- This is one nondeterministic Bedrock run, not a statistically stable benchmark result.
- The run was generated from a dirty worktree; reproducibility should be anchored to the commit that includes this scorecard and the associated code changes.
- The restaurant environment remains fake and deterministic.
- Judges are scenario-local and partly heuristic.
- The run supports the evaluation construct and exposes architecture-sensitive failure modes, but it is not proof that graph memory generally beats transcript context.
- Because first-party LLM agents now share response-time instructions, this scorecard should supersede older Bedrock scorecards for current architecture comparisons.
