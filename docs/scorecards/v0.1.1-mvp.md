## Key Observations

- This scorecard demonstrates the core FidelityBench construct: stateless recall burden versus rule-memory intention fidelity.
- `StatelessAgent` fails by asking the user to repeat context that was already established: cuisine, budget, dietary constraints, and location.
- `RuleMemoryAgent` asks only for the genuinely missing field, party size, then holds an available restaurant that matches the accumulated dinner intent.
- The run also exposes a useful diagnostic: `RuleMemoryAgent` succeeds at the final task but still loses query-fidelity points because its `restaurants.search` arguments do not fully encode the remembered constraints.

## Failure Modes

- Recall burden: `StatelessAgent` shifts known context back onto the user.
- Non-completion: `StatelessAgent` does not search or place a reservation hold.
- Query translation gap: `RuleMemoryAgent` completes the task but does not express all remembered constraints in the search call.

## Limitations

- These are deterministic baselines, not live product agents.
- The restaurant tool environment is fake and deterministic.
- Judges are scenario-local and partly heuristic.
- This run proves the evaluation construct, not broad benchmark validity.
- This run is not yet proof that graph memory beats transcript context.
