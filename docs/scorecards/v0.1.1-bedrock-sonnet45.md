## Key Observations

- `TranscriptLLMAgent` is the strongest LLM baseline in this run: 390 total, with perfect scores on board update, reflection, and Alex pushback.
- `BlockMemoryLLMAgent` is close overall at 379 and is the only LLM baseline to complete the dinner reservation flow in this run, scoring 110/110 on dinner.
- Boundary fidelity is a real discriminator: `BlockMemoryLLMAgent` and `FileMemoryLLMAgent` both leaked the private staffing concern in `board_update_privacy_001`, while transcript and windowed transcript preserved that boundary.
- The action/tool-loop scenarios remain uneven. Transcript, windowed transcript, and file memory often translated memory into `restaurants.search` arguments but failed to complete successful holds.
- `StatelessLLMAgent` does surprisingly well on some writing tasks because the final prompt contains enough local task shape to produce a plausible draft, but it still loses memory-specific dimensions and recall-burden points.
- `FileMemoryLLMAgent` is now a valid Bedrock baseline in the full suite, after the message-order fix.

## Failure Modes

- Tool-loop incompletion: transcript and file memory searched on dinner but did not complete the hold.
- Temporal action failure: no LLM baseline completed the supersession scenario; transcript, windowed transcript, and file memory did encode the latest Mexican/Mission/$50/vegetarian constraints in search args.
- Boundary leak: block memory and file memory leaked the private staffing concern in the board-update scenario.
- Recall burden under limited context: windowed transcript and file memory asked for known dinner time, and stateless asked for multiple known constraints.
- Structured-memory extraction gap: block memory failed temporal supersession in this run, suggesting either extraction, supersession deletion, or response use of memory needs closer diagnosis.

## Limitations

- This is one nondeterministic Bedrock run, not a statistically stable benchmark result.
- The restaurant environment remains fake and deterministic.
- Judges are scenario-local and partly heuristic.
- The run was generated from a dirty worktree; reproducibility should be anchored to the commit that includes this scorecard and the associated code changes.
- The run supports the evaluation construct and exposes architecture-sensitive failure modes, but it is not yet proof that graph memory beats transcript context.
